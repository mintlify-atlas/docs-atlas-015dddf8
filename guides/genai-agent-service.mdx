## Overview

The `GenAIAgentService` class simplifies the creation of streaming endpoints for ADK agents in Next.js API routes. It handles message validation, streaming response generation, tool execution, and error handling.

<Info>
  This service provides native support for Server-Sent Events (SSE) and is fully compatible with the Vercel AI SDK.
</Info>

## Basic Usage

Here's a complete example of a Next.js API route using `GenAIAgentService`:

```typescript route.ts
import { GenAIAgentService, OllamaModel } from '@yagolopez/adk-utils';
import { LlmAgent } from '@google/adk';

export async function POST(req: Request) {
  // Get user messages from the frontend
  const { messages } = await req.json();
    
  // Create instance of ADK agent  
  const agent = new LlmAgent({
    name: 'my-agent',
    model: new OllamaModel('qwen2.5:0.5b', 'https://ollama.com'),
    description: 'Agent description',
    instruction: 'You are a helpful assistant...',
    tools: [],
  });
  
  // Create agent service to invoke the agent in the endpoint          
  const service = new GenAIAgentService(agent);

  // Check if the messages sent by the user are valid
  if (!service.validateMessages(messages)) {
    return GenAIAgentService.createErrorResponse('Invalid messages', 400);
  }

  // Invoke the agent with messages from the user and return the agent response in stream format
  return service.createStreamingResponse(messages);
}
```

## Constructor

The service is initialized with an ADK agent:

```typescript
const service = new GenAIAgentService(agent: LlmAgent)
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `agent` | `LlmAgent` | An ADK LlmAgent instance with model, tools, and configuration |

## Core Methods

### validateMessages()

Validates incoming messages before processing:

```typescript
validateMessages(messages: UIMessage[] | undefined): boolean
```

**Returns:** `true` if messages are valid (array with at least one message), `false` otherwise.

<Tabs>
  <Tab title="Usage">
    ```typescript
    const { messages } = await req.json();

    if (!service.validateMessages(messages)) {
      return GenAIAgentService.createErrorResponse('Invalid messages', 400);
    }
    ```
  </Tab>
  <Tab title="What it checks">
    - Messages is not `undefined`
    - Messages is an array
    - Array has at least one message
  </Tab>
</Tabs>

### createStreamingResponse()

Generates a streaming Response object with Server-Sent Events:

```typescript
createStreamingResponse(messages: UIMessage[]): Response
```

**Returns:** A `Response` object with `text/event-stream` content type.

<Steps>
  <Step title="Transform Messages">
    Converts Vercel AI SDK `UIMessage` format to ADK `Content` format
  </Step>
  <Step title="Format Tools">
    Prepares agent tools for the LLM request
  </Step>
  <Step title="Stream Response">
    Creates a ReadableStream that:
    - Sends SSE start events
    - Streams model responses
    - Executes tools if needed
    - Handles multi-turn conversations
    - Sends SSE end events
  </Step>
  <Step title="Return Response">
    Returns the streaming Response object
  </Step>
</Steps>

### createErrorResponse() (static)

Creates a JSON error response:

```typescript
static createErrorResponse(
  message: string, 
  status: number, 
  details?: string
): Response
```

**Example:**

```typescript
if (!messages) {
  return GenAIAgentService.createErrorResponse(
    'Missing messages',
    400,
    'Request body must include a messages array'
  );
}
```

## Message Transformation

The service automatically transforms messages between formats:

<Accordion title="UIMessage to Content Transformation">
  **Input (Vercel AI SDK):**
  ```typescript
  {
    role: 'user',
    id: '123',
    parts: [{ type: 'text', text: 'Hello' }]
  }
  ```

  **Output (ADK):**
  ```typescript
  {
    role: 'user',
    parts: [{ text: 'Hello' }]
  }
  ```

  The transformation:
  - Maps `'user'` → `'user'`
  - Maps `'assistant'` → `'model'`
  - Extracts text from parts
  - Removes UI-specific fields (id, etc.)
</Accordion>

## Tool Execution

The service handles tool execution automatically in a multi-turn conversation loop:

```typescript
import { GenAIAgentService, OllamaModel } from '@yagolopez/adk-utils';
import { LlmAgent, Tool } from '@google/adk';

// Define a tool
const weatherTool = new Tool({
  name: 'get_weather',
  description: 'Get current weather for a city',
  parameters: {
    type: 'OBJECT',
    properties: {
      city: { type: 'STRING', description: 'City name' }
    },
    required: ['city']
  },
  func: async ({ city }) => {
    // Your tool logic here
    return { temperature: 72, condition: 'sunny' };
  }
});

export async function POST(req: Request) {
  const { messages } = await req.json();
  
  const agent = new LlmAgent({
    name: 'weather-agent',
    model: new OllamaModel('qwen2.5-coder'),
    instruction: 'Help users check the weather.',
    tools: [weatherTool],  // Add tools here
  });
  
  const service = new GenAIAgentService(agent);
  
  if (!service.validateMessages(messages)) {
    return GenAIAgentService.createErrorResponse('Invalid messages', 400);
  }
  
  // Tool execution happens automatically inside createStreamingResponse
  return service.createStreamingResponse(messages);
}
```

<Note>
  The service executes up to 5 turns (configurable via `MAX_TURNS`) to handle complex multi-step tool interactions.
</Note>

## Error Handling Patterns

### Client-Side Errors

```typescript
export async function POST(req: Request) {
  try {
    const { messages } = await req.json();
    
    if (!service.validateMessages(messages)) {
      return GenAIAgentService.createErrorResponse('Invalid messages', 400);
    }
    
    return service.createStreamingResponse(messages);
  } catch (error) {
    console.error('Request parsing error:', error);
    return GenAIAgentService.createErrorResponse(
      'Invalid request format',
      400,
      error instanceof Error ? error.message : 'Unknown error'
    );
  }
}
```

### Server-Side Errors

```typescript
export async function POST(req: Request) {
  try {
    const { messages } = await req.json();
    const service = new GenAIAgentService(agent);
    
    if (!service.validateMessages(messages)) {
      return GenAIAgentService.createErrorResponse('Invalid messages', 400);
    }
    
    return service.createStreamingResponse(messages);
  } catch (error) {
    console.error('Server error:', error);
    return GenAIAgentService.createErrorResponse(
      'Internal server error',
      500,
      process.env.NODE_ENV === 'development' ? String(error) : undefined
    );
  }
}
```

<Warning>
  Errors during streaming are automatically caught and sent to the client as text in the stream with the format: `❌ System Error: [error message]`
</Warning>

## Advanced Patterns

### Middleware Pattern

```typescript
import { GenAIAgentService } from '@yagolopez/adk-utils';
import { NextRequest } from 'next/server';

// Auth middleware
function withAuth(handler: Function) {
  return async (req: NextRequest) => {
    const token = req.headers.get('authorization');
    if (!token) {
      return GenAIAgentService.createErrorResponse('Unauthorized', 401);
    }
    return handler(req);
  };
}

// Rate limiting middleware
function withRateLimit(handler: Function) {
  return async (req: NextRequest) => {
    const ip = req.ip || 'unknown';
    // Check rate limit logic
    if (isRateLimited(ip)) {
      return GenAIAgentService.createErrorResponse('Too many requests', 429);
    }
    return handler(req);
  };
}

// Compose middleware
const POST = withAuth(withRateLimit(async (req: Request) => {
  const { messages } = await req.json();
  const service = new GenAIAgentService(agent);
  
  if (!service.validateMessages(messages)) {
    return GenAIAgentService.createErrorResponse('Invalid messages', 400);
  }
  
  return service.createStreamingResponse(messages);
}));

export { POST };
```

## SSE Format

The service emits Server-Sent Events in the following format:

```typescript
// Start events
data: {"type":"start"}\n\n
data: {"type":"start-step"}\n\n
data: {"type":"text-start","id":"text-1"}\n\n

// Text deltas (actual content)
data: {"type":"text-delta","id":"text-1","delta":"Hello"}\n\n
data: {"type":"text-delta","id":"text-1","delta":" world"}\n\n

// End events
data: {"type":"text-end","id":"text-1"}\n\n
data: {"type":"finish-step"}\n\n
data: {"type":"finish","finishReason":"stop"}\n\n
```

See the [Streaming guide](/guides/streaming) for more details on the SSE format.
